# Полнотекстовый поиск по файлам проекта

Данный проект представляет собой реализацию простейшей системы
[полнотекстового поиска](https://en.wikipedia.org/wiki/Full-text_search)
по файлам с исходным кодом предыдущих лабораторных работ.

В основе решения лежит построение
[инвертированного индекса](https://en.wikipedia.org/wiki/Inverted_index),
который позволяет эффективно находить документы, содержащие слова из поискового запроса.
Индекс подготавливается заранее и сохраняется в виде набора файлов со структурами данных,
оптимизированными для поиска.

Проект состоит из двух отдельных приложений:

## Индексатор

Приложение индексатора выполняет:
- обход директории с исходными файлами;
- токенизацию текстового содержимого документов;
- построение инвертированного индекса (термин → список документов);
- сбор статистики для ранжирования (частоты терминов, длины документов и др.);
- сохранение поискового индекса на диск.

## Поисковое приложение

Поисковое приложение:
- загружает ранее построенный индекс;
- принимает текстовый запрос пользователя;
- находит релевантные документы с использованием инвертированного индекса;
- выполняет ранжирование результатов с помощью алгоритма
  [BM25](https://en.wikipedia.org/wiki/Okapi_BM25),
  основанного на метрике
  [TF-IDF](https://en.wikipedia.org/wiki/Tf%E2%80%93idf);
- выводит документы, отсортированные по степени релевантности.

## Цель проекта

Целью работы является изучение базовых принципов построения поисковых систем,
включая индексирование текстов, обработку поисковых запросов и ранжирование документов.
Проект демонстрирует упрощённую модель работы современных поисковых движков.

## Функциональность

### Функциональность индексирующей программы

- Консольное приложение.
- Обходить все файлы по определенной директории рекурсивно
- Осуществлять подготовку инвертированного индекса и сопутствующий файлов
- Приведение слов к [нормальной форме](https://ru.wikipedia.org/wiki/%D0%9B%D0%B5%D0%BC%D0%BC%D0%B0%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F) не требуется

### Функциональной поисковой программы

- Консольное приложение, взаимодействие с пользователем через стандартные потоки ввода и вывода
- Осуществлять поиск по построенному индексу
- Ранжировать найденные документы согласно BM25
- Выдавать в качестве результат список файлов с номерами строк где встречается данное слово
- Поддерживать синтаксис запросов с 'AND' и 'OR'
- Некорректный запрос должен приводить к выводу ошибки
- Поиск регистронезависимый

### Синтаксис запросов

Должен поддерживать скобки а также операции AND и OR (регистр имеет значение).
Таким образом в качестве запроса выступают логические выражения. Разделитель между словами - пробел(ы)

### Примеры 
 - "for"
 - "vector OR list"
 - "vector AND list"
 - "(for)"
 - "(vector OR list)"
 - "(vector AND list)"
 - "(while OR for) and vector"
 - "for AND and"
